显现形态|display

Name: display
Value: [ <display-outside> || <display-inside> ] | <display-listitem> | <display-internal> | <display-box> | <display-legacy>
Initial: inline
Applies to: all elements
Inherited: no
Percentages: n/a
Media: all
Computed value: as specified
Animatable: no

display属性定义了一个盒子的显现类型，其中包括两种基础的类型：

  • inner display类型 ，这定义了(如果是非替换元素)它将产生出何种格式化上下文以及表现出它的后代盒子的布局方式。 (替换元素的inner display属性不在CSS范畴内)

  • outer display类型 ，这规定了一个盒子在其父级格式化上下文的中表现的形式。

<display-outside>  = block | inline | run-in ;
<display-inside>   = flow | flow-root | table | flex | grid | ruby ;
<display-listitem> = list-item && <display-outside>? && [ flow | flow-root ]?
<display-internal> = table-row-group | table-header-group |
                                         table-footer-group | table-row | table-cell |
                                         table-column-group | table-column | table-caption |
                                         ruby-base | ruby-text | ruby-base-container |
                                         ruby-text-container ;
<display-box>      = contents | none ;
<display-legacy>   = inline-block | inline-list-item |
                                         inline-table | inline-flex | inline-grid ;

outer display类型。属性值block，会使元素成为一个块级盒子。属性值inline，会使得元素称为一个行内盒子。

innter display类型。属性值flow，决定了元素如何布局其内容。属性值flex，当元素生成一个主要的弹性容器和实现弹性格式化上文时，设置此属性值。

属性值block，默认父元素宽度，可以设置宽高，本身独占一行（换行）。 div, p, h1~6, ul, form,...默认block属性值。

inline，默认文本内容宽度，不可设置宽高，相对于其前面的行内元素同行显示（同行），如果内容超过一行，在元素内换行。 span, a, label, cite, em,...默认inline属性值。

inline-block，默认文本内容宽度，可以设置宽高，相对于其前面的行内/行内块级元素同行显示（同行），如果内容超过一行，整块元素换行。 input, textarea, select, button,...默认inline-block属性值

另外，inline和inline-block，都可设置行内元素的vertial-align属性。

none，不显示元素，不占用页面空间。visibility: hidden，不显示元素，但仍然占用页面空间。

flex，设置一个容器为弹性容器，而容器中在文档流中的直接子元素就是弹性子元素/弹性项。position后的就不是弹性项。

浮动|float

Name: float
Value: left | right | top | bottom | start | end | none | <page-floats>
Initial: none
Applies to: all, but see text
Inherited: no
Animatable: no
Percentages: N/A
Media: visual
Computed value: as specified
Canonical order: order of grammar

设置为浮动的元素，默认宽度为内容宽度。多个浮动元素会相互紧靠不会有定位元素的堆叠效果。

设置float的元素会改变元素的display属性，变成inline-block。可以设置宽高，并且自动换行，当然如果宽高都大于容器，那么还是会超出的（前提是overflow:visible;），一般来讲是超出的右边和下边。设置浮动的元素实际上是盒子本身对其他盒子没有边界的“排外”效果（常说的不在文档流中），但是盒子对其他盒子里的内容会“排外”。所以就有了一个浮动元素A跟随着一个非浮动未clear的元素B，B的框会忽视A的框按照块级流方向一个接一个排到（通常）左上角，但是B的内容任然是被“排外”，如果B的后面也有个非浮动未clear的元素C且B框没有填满A框缺失的部分，那么C的框也会继续填上去，但是C的内容也是被“排外”了。这里的内容是不会相互“排外”，所以B和C的内容可能会重叠。总之，浮动元素的盒子跟非浮动元素的内容排斥，跟非浮动元素的盒子亲和；非浮动元素的内容之间是相互亲和的。

清除浮动|clear

Name: clear
Value: none | left | right | both
Initial: none
Applies to: block-level elements
Inherited: no
Animatable: no
Percentages: N/A
Media: visual
Computed value: as specified
Canonical order: N/A

clear属性，用于清除前面元素的浮动效果的延续对后续元素的影响。 both，左右清除浮动。 left，清除左边浮动。 right，清除右浮动。

应用于浮动元素的后续元素，一般是块级元素。 通常在浮动元素后面增加一个空白的块级元素并且设置clear:both;属性来清除浮动。

定位|position

Name: position
Value: static | relative | absolute | sticky | fixed
Initial: static
Applies to: all elements except table-column-group and table-column
Inherited: no
Percentages: N/A
Media: visual
Computed value: specified value
Animatable: no

position属性确定了使用何种定位算法来计算一个盒子的位置。

定位最重要的一点就是：设置了参照物。 设置了positon属性后的元素，可以通过设置top,right,bottom,left,z-index来定位。 top,right,bottom,left是用来设置元素边缘距离参照物的边缘的距离。 z-index设置元素在z轴上的排序。但是因为元素还有一个堆叠/栈的概念，所以并不一定z-index设置的越大，元素就一定堆在最上面。 堆叠/栈，是指在未特别设置z-index时，元素按文本流的顺序，越往后的元素，会堆在越上面。而如果2个元素有各自的父元素，无论两个子元素如何设置z-index，处于z-index设置大的父元素里的子元素，永远要堆在z-index设置小的父元素里的子元素上面。因为z-index只是相对于父元素的z轴位移。

main
z-index 10
z-index 9

默认position: static，以正常流的方式布局。只有设置此属性值的元素不可用top、right、bottom和left属性。

position: relative; 元素仍在文档流中，不会对其他元素造成影响，即元素移动后其原来的位置仍然会为它空出来。 参照物为元素本身。 常用于元素在文档流中的层级。 最重要的使用场景是最为子元素绝对定位时给父元素设置相对定位。

position: absolute; 一个元素设置绝对定位后,它的宽度默认变为内容宽度。 元素会脱离文档流，相当于浮到了文档流上面。 绝对定位的参照物为离它最近的设置了相对定位的祖先元素或者根元素，需要先对参照物设置相对定位。一个在容器中定位的子元素如果未设置宽高，设置top/right/bottom/left会撑开子元素的盒子。

instance

注意：非根元素设置relative或static，它的参照物容器边界是，这个容器的内容边界；而如果非根元素设置absolute而且作为容器的祖先元素是非static的，则若这个祖先是块级元素，参照物容器边界是边框内边界，若这个祖先是行内元素，参照物容器边界就是内容边界。
