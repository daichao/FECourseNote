简写字体|font

Name: font
Value: [ [ <'font-style'> || <font-variant-css21> || <'font-weight'> || <'font-stretch' ]? <'font-size'> [ / <'line-height'> ]? <'font-family'> ] | caption | icon | menu | message-box | small-caption | status-bar
Initial: see individual properties
Applies to: all elements
Inherited: yes
Percentages: see individual properties
Media: visual
Computed value: see individual properties
Animatable: see individual properties

font是以上所有属性的简写shorthand形式。

当使用开发者自定义简写字体时，font-size和font-family必须出现而且要求按照先后顺序，否则设置font将失效。当行高line-height一同写入font时，line-height属性值必须写在font-size属性值之后，且要用斜杠/隔开。

p { font: 12px/2 "Microsoft YaHei", serif; }

注意：使用简写shorthand会使省略的部分属性值都会重置为默认值。所以不要在单个字体属性值设置后，又把简写字体属性font放于最后。

使用系统字体

使用系统字体，会通过获取系统某些部件的字体大小、字体系列、习题加粗、字体风格和字体变形等默认设置，并将其应用到某个元素上。而不需要再对每个属性进行设置，一旦设置了系统设置，可以通过单个字体属性对字体进行设置。

  1. caption

    用于有标签的控件

    p { font: caption; }
  2. icon

    用于对图标加标签

    p { font: icon; }
  3. menu

    用于菜单，即下拉菜单和菜单列表

    p { font: menu; }
  4. message-box

    用于对话框

    p { font: message-box; }
  5. small-caption

    用于对小控件加标签

    p { font: small-caption; }
  6. status-bar

    用于窗口状态条

    p { font: status-bar; }

行高|line-height

Name: line-height
Value: normal | <number> | <length> | <percentage> | inherit
Initial: normal
Applies to: all elements
Inherited: yes
Percentages: refers to the font size of the element itself
Media: visual
Computed value: for <length> and <percentage> the absolute value; otherwise as specified
Animatable: yes, as a number, a length
Canonical order: the unique non-ambiguous order defined by the formal grammar

在应用到块级元素时，line-height定义了元素中文本基线之间的最小距离。注意，它定义的是最小距离，而不是一个绝对数值，文本基线拉开的距离可能比line-height值更大。

文本行中的每个元素都会生成一个内容区，这由字体的大小确定。这个内容区则会生成一个行内框，如果不存在其他因素，这个行内框就完全等于该元素的内容区，由line-height产生的行间距就是增加或减少各行内框高度的因素之一。

要确定一个给定元素的行间距，只需要将line-height的计算值减去font-size的计算值。这个值是总得行间距。而且行间距可能是负的。然后再除2，将行间距的一般分别应用到内容区的顶部和底部。其结果就是该元素的行内框。

对于line-height的属性值，px是直接设置，em和百分比是继承父类的line-height计算后的值，纯数值是只继承父类的数值(缩放因子)而后自行计算。都是相对于字号而言，px在字号变大时虽然行高不变行但间距会缩小，em、百分比和纯数值都是随着字体大小变化的，只不过要考虑到继承的差异。总而言之，要让父类的行高被继承下去，父类就用em、ex和百分比；只让父类相对字号的比例被继承，父类就用作为缩放因子的纯数值。对于设置在元素本身的行高，只要相对字体大小font-size进行计算就好了。

normal，元素不会对双向算法打开附加的一层嵌套。对于行内元素，顺序的隐式重拍会跨元素边界进行。

embed，如果是一个行内元素，这个值对于沙un共享算法会打开附加的一层嵌套。这个嵌套层的方向由direction属性指定。

bidi-override，这会为行内元素创建一个覆盖。对于块级元素，将为不在另一块中的行内后代创建一个覆盖。

Name: white-space
Value: normal | pre | nowrap | pre-wrap | pre-line
Initial: normal
Applies to: all elements
Inherited: yes
Percentages: N/A

Animatable: no

以下是各属性值介绍：

属性值 描述
normal 作为white-space的初始值。浏览器在处理HTML文档时会把所有连续的空白符合并为一个空格，把强制换行符转换为一个空格。当然，如果一行的文本太多，仍旧会自动换行显示，但是这跟使用强制换行符有本质区别。
pre 这个值会保留多余的空白符和强制换行符，所以对于设置了white-space: pre;的元素相当于拥有了HTML中pre元素的效果。并且不允许自动换行。
nowrap 这个值像normal一样会合并强制换行符、空白符和缩进符，又像pre不允许自动换行。除非使用br元素，否则设置这个值会防止元素中的文本换行。常与overflow:hidden;和text-overflow: ellipsis;一起使用，来达到文本溢出时的省略效果。
pre-wrap 这个值像pre会保留多余的强制换行符、空白符和缩进符，又像normal允许自动换行。
pre-line 这个值像normal会合并空白符和缩进符，且允许自动换行。但是会保留强制换行符。

下表总结了white-space的不同值的行为：

值\行为 强制换行符(New Lines) 空白符和缩进符(tabs) 自动换行(Text Wrapping)
normal 合并 合并 是
pre 维持 维持 否
nowrap 合并 合并 否
pre-wrap 维持 维持 是
pre-line 维持 合并 是

单词截断|word-break

Name: word-break
Value: normal | keep-all | break-all
Initial: normal
Applies to: all elements
Inherited: yes

Media: visual

Animatable: no

这个属性用于设置字母之间的自动软换行。

normal根据文本在行内的使用情况截断单词。

break-all对于任何字母之间都可进行截断和换行。

keep-all所有连在一块的单词都不截断。
